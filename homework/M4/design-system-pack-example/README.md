# Design System Pack — пример ZIP-паттерна

> Это распакованный пример экспортного пакета дизайн-системы (JSON + CSS + MD), упакованного в ZIP для передачи AI-агенту как единый контекст.

---

## Что это

Три файла, которые AI-агент получает как единый контекст для генерации UI:

| Файл | Формат | Для кого |
|---|---|---|
| [`design-system.json`](./design-system.json) | JSON | Программный доступ, парсинг токенов, Tailwind config |
| [`design-system.css`](./design-system.css) | CSS | Прямое использование, импорт в проект |
| [`design-system.md`](./design-system.md) | Markdown | AI-агент: философия + готовый промпт + ссылки |

Все три файла лежат в одной папке. Это принципиально — относительные ссылки в Markdown работают только в этом случае.

---

## Зачем три формата

Каждый формат решает отдельную задачу.

**JSON** — source of truth для токенов. Удобно парсить скриптами, генерировать из него Tailwind config, использовать в CI/CD. Содержит весь дизайн в структурированном виде: цвета, типографику, spacing, состояния, компоненты.

**CSS** — готовый к использованию код. Можно импортировать в проект как есть. CSS-переменные в `:root`, классы компонентов, поддержка light/dark mode, accessibility utilities. Агент может читать его как reference и воспроизводить паттерны в нужном фреймворке.

**Markdown** — инструкция для AI-агента. Самый важный файл. Содержит то, чего нет в токенах: дизайн-философию (вербальный «характер» интерфейса), конкретный промпт для применения системы, ссылки на JSON и CSS. Без философии агент работает по числам, не по характеру — и выдаёт generic результат.

---

## Зачем ZIP при передаче

При скачивании нескольких файлов браузер показывает запрос «разрешить скачивание нескольких файлов». Если пользователь нажимает «Отклонить» — ничего не скачается, и он не поймёт почему.

ZIP решает это одним скачиванием. После распаковки все три файла оказываются в одной папке — и относительные ссылки в Markdown (`./design-system.json`, `./design-system.css`) продолжают работать. Это не техническая мелочь, а архитектурное решение, обеспечивающее связность пакета.

---

## Как использовать

1. Скопируй папку в корень своего проекта.
2. Открой `design-system.md` — прочитай философию и скопируй готовый промпт.
3. В начале задачи AI-агенту: вставь промпт из `design-system.md` + укажи, что `design-system.json` и `design-system.css` лежат рядом как reference.
4. Агент использует эти файлы не как готовый код, а как спецификацию — и адаптирует под фреймворк проекта (Tailwind / styled-components / vanilla CSS / shadcn/ui).

## Почему паттерн универсальный

Подход разрабатывался для дизайн-систем, но структура обобщается на любой AI-редактор генерируемого контента:

| Домен | JSON | CSS / исходник | Markdown |
|---|---|---|---|
| Дизайн-система | design tokens | ready CSS | философия + промпт |
| Резюме | структура секций | шаблон PDF | инструкция по тону |
| Лендинг | блоки + токены | базовые стили | messaging + промпт |
| Презентация | слайды + данные | мастер-слайд | нарратив + промпт |
| RAG bundle | конфиг источников | — | инструкция агента |

Принцип один: упаковать machine-readable данные + human-readable инструкцию + готовый стартовый код в один архив с относительными ссылками.
